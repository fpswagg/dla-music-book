# Phase 11: i18n (Internationalization)

## Goal
Make the app UI translatable in French, English, and Duala (placeholder) using next-intl.

## Setup
- Locale detection from URL prefix: /en/*, /fr/*, /duala/*
- Default locale: fr (French)
- Middleware rewrites for locale routing
- Language switcher component in header

## Message Files (src/i18n/messages/)
Create en.json and fr.json with keys for all UI strings:
- nav.home, nav.songs, nav.collections, nav.dashboard, nav.admin
- home.title, home.subtitle, home.search_placeholder, home.featured, home.browse_all
- songs.title, songs.search_placeholder, songs.no_results, songs.filters
- song.by_author, song.version, song.lyrics, song.annotations, song.notes, song.share, song.print, song.like
- auth.login, auth.register, auth.email, auth.password, auth.phone, auth.or_continue_with
- dashboard.my_likes, dashboard.my_collections, dashboard.create_collection
- admin.overview, admin.songs, admin.users, admin.tags, admin.authors, admin.analytics, admin.backups
- common.loading, common.error, common.save, common.cancel, common.delete, common.edit, common.search
- status.draft, status.finished
- mock.banner_message

## Duala Placeholder (src/i18n/messages/duala.json)
- Copy structure from en.json
- Fill with English values + comment that translations needed
- Create docs/INTEGRATION-DUALA.md explaining how to replace

## Note
Song content is NOT translated through i18n. Songs have their own language field (Duala, French, English) which is a property of the song itself.
